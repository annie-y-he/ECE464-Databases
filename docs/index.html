PROJECT NAME


SQL: Solid Quality Library

DESCRIPTION


SQL (Solid Quality Library) is a digital library for ebooks developed by DB.jar, designed to enhance access to a wide array of digital reading materials across various genres and disciplines. Launched with the aim of democratizing the availability of educational and recreational resources, SQL distinguishes itself with a user-centric design that not only supports online reading but also facilitates cross-platform annotation syncing. The platform encourages community participation by allowing users to upload and manage ebook content, thereby continuously enriching its collection.

PURPOSE


The platform's purpose is to provide an easy-to-use, comprehensive repository of eBooks that supports educational and recreational reading needs. It encourages knowledge sharing and collaboration among users, fostering a vibrant community of readers and contributors.

General Functionality


Database Functionality
Books: Manages data related to eBooks, including metadata for titles, authors, publication dates, and genres.
Users: Handles user information, including authentication data, profiles, and preferences.

Backend Functionality
Security: Ensures the application's data and user interactions are secure, managing authentication, authorization, and data protection.
Serving: Handles the logic and infrastructure needed to serve application content to users, including API management, request routing, and content delivery.

Frontend Functionality
GUI: Focuses on designing and implementing the user interface, ensuring it is intuitive, engaging, and accessible across different devices.
Online Reading: Provides a rich online reading experience with features that allow users to easily read, navigate, and interact with eBooks.

Additional Functionality
OCR: Converts images of text within eBooks into searchable and editable text, enhancing accessibility and functionality.
Annotation: Enables users to add personal notes, highlights, and bookmarks to eBooks, facilitating a more interactive reading experience.
Social: Integrates features that allow users to share books, annotations, or reading lists with others, promoting community engagement.
Recommendation: Utilizes user data to suggest relevant eBooks, enhancing content discoverability and personalizing the user experience.
Collection: Allows users to organize eBooks into personalized collections or lists, making it easier to manage and access preferred content.

Types of Users


Reader: Can read, collect, annotate books.

Contributor: In addition to readers, contributors can upload eBooks and manage (delete, modify metadata) their contributions.

Manager: In addition to contributors, managers can manage all books in the library.

Administrator: In addition to managers, administrators can assign roles to a specific user.


* The default role is Reader. During account creation, a user can request a role, which will be reviewed by an Admin.

User Views


Reader:
Profile: reviews or alter personal information. See other users' public information.
Library: able to view all books with the number of editions it has.
Book: see name and description of a book. And see a list of its publications (cover and edition name), tags, authors, languages
Publication: still see name, description, tags of the book. Author list adds author specific to this edition. Also shows the path to the actual file
Collection: see different collections. Public vs private, self vs other. The publications one has annotated.

Contributor:
Upload: see and manage the books one has uploaded

Manager:
Manage: able to edit all metadata.

Administrator:
User: able to see all users and assign them roles.

Data Description


Publication (publication):
Stores information about publications, such as books or articles.
Includes fields for the publication ID (pid), International Standard Book Number (isbn), file path (file), edition, number of pages, and cover image.

Book (book):
Contains details about books.
Stores the book ID (bid), description, and name of the book.

Tag (tag):
Stores tags or labels that can be assigned to books.
Each tag is uniquely identified by its name (tname).

Series (series):
Contains information about book series.
Stores the series ID (sid) and name of the series.

Author (author):
Stores information about authors.
Includes the author ID (aid), name, description, date of birth (dob), and date of death (dod).

Publisher (publisher):
Contains information about publishers.
Stores the publisher name (pname).

Language (language):
Stores information about languages.
Each language is uniquely identified by its name (lname).

User (user):
Stores user information for authentication and personalization.
Includes the user ID (uid), password, date of birth (dob), email, username, first name, and last name.

* The database will be populated with data from user input, existing databases, filesystem, etc.
